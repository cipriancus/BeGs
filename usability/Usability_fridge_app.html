Usability Test Plan Section

Test Plan for: Begs Fridge app Component

Persona Type Tested: Primary, Secondary, Negative

Context: A meeting room from the office for the primary, and at home for the rest.
Duration:15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • What is the user's opinion about the main page from Begs fridge application
  • How intuitive the wish list is
  • How difficult is it to navigate to the product detail page?
Scenario

For the first scenario, users will interact with the simulated refrigerator application in a browser through a web application. The interaction will be small because with the refrigerator the user will have the possibility to choose from the list of products the food they want and put them in the wish list. He can see product details and give up a product he no longer wants.

Tasks
  • How can you select a product in your wish list? 30sec
  • How can you see new products? 40sec
  • How can you view details about a product? 50sec
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • What was the first impression when you saw the refrigerator screen?
  • Was it difficult to view the details of a particular product?
  • Did you feel in difficulty finding the new product list?
  • What seemed the easiest to you?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Other Notes
  • Users who do not have smart devices in their homes have been skeptical about testing this type of product.

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • When 50% of users had a quick interaction without any difficulty with the refrigerator application
  • When 50% of users easily add desired products to wish list.
  • When 50% of users entered in less than 60 seconds to see details about a product.

Summary

  • Finding 1: User 3 finds it difficult to use the app as the font cannot increase.
  • Finding 2: All 4 users successfully completed all tasks.
  • Finding 3: A no of 2 users feel the app is not fully modern.
  • Finding 4: A no of 2 users feel the coloring schema can be more lively.

Post-test Questions Answers
  • Q:What was the first impression when you saw the refrigerator screen? A: It seems simple, 2 columns of products.
  • Q:Was it difficult to view the details of a particular product? A: No, simply press on the card.
  • Q:Did you feel in difficulty finding the new product list? A: No, it was right in front of me.
  • Q:What seemed the easiest to you? A: Adding to shopping list, it was a swipe.
Test Observations
  • User 3 had difficulties in seeing the font as she wore grasses, a mechanism of increasing the font size would suffice.
  • User 1 and 4 believed the design can be improved with more recent android material design components.
  • User 1 and 4 believed that the app needs to have a more colorful palette.
  • Users need some time to accommodate with the concept and app but they understand what the purpose is and how to use it.
Potential Design Improvements
  • Improve design color schema
  • Make the app more up to material design
  • Add mechanism for increasing font size
