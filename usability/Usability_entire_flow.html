Test Plan for: Begs Full flow, smartphone, fridge, smarwatch Component

Persona Type Tested: Primary

Context: A meeting room from the office
Duration:30 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • What is the user's opinion about the main page from Begs smartwatch application
  • How intuitive the navigation of the shopping list is
Scenario

The users will be given the smartwatch, the phone and the test fridge user interface. The goal is to add a product from the fridge that he thinks he needs to buy, see the list on the smartwatch and create the gestures and on the phone to see the same list.

Tasks
  • How can you add a product in the shopping list on the fridge? 30sec
  • Can you see the same shopping list from the fridge on the smartwatch? 60sec
  • How can you confirm that you bought a specific product on the smartwatch? 60sec
  • Can you see the updated shopping list from the smartphone on the phone? 60sec
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • How difficult is this flow?
  • What was the first impression when you had to use the three devices together?
  • Do you feel there is a way we can improve the flow?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • When the users can complete all tasks.
  • He understands the concept between the three apps.

Summary

  • Finding 1: Users are first frightened by the task complexity.
  • Finding 2: Users feel they do not have enough information to complete the task - e.g in tutorial or automatic guidance from the app.
  • Finding 3: They have completed the tasks with some intuition.
  • Finding 4: They understand and like the concept once they've done it once or twice.

Post-test Questions Answers
  • Q:How difficult is this flow? A:At first the task seemed a bit daunting but it's fairly easy once you get the hang of it.
  • Q:What was the first impression when you had to use the three devices together? A:Interesting but I didn't know from where to start.
  • Q:Do you feel there is a way we can improve the flow? A:A better tutorial, in the start of the app and in-app suggestions.
Test Observations
  • Users familiar with technology were able to complete the tasks successfully.
  • There is a need to further enhance the application tutorial and create in app suggestions.
