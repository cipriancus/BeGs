Usability Test Plan Section

Test Plan for: Begs Mobile app Component

Persona Type Tested: Primary, Secondary, Negative

Context: A meeting room from the office for the primary, and at home for the rest.
Duration:15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • What is the user's opinion about the Begs mobile application.
  • How intuitive the recommendations are.
Scenario

In the first scenario, the user has the first interaction with the mobile application. This first interaction will have a welcome page, she/he will can select his/her preferences or/and allergies and after that will be redirect to the main page. Will find the main categories and recommendations used, as well as the main menu of the application. It will also have a product selected, scanned with QR code and added to the shopping cart to complete the order. During this flow, you will find the moving image.

Tasks
  • How can you choose preferences or allergies? 25sec ;
  • On what ranks is the "Recommendations" category? 30sec ;
  • How do you find the recommendations for products? 30sec ;
  • Which image on the main page is in motion? 50sec ;
  • How can you see new products? 40sec ;
  • How can you scan a QR code product? 40sec ;
  • How can you add a product to your shopping cart? 60 sec ;
Post-test Questions

These questions could be presented to the user as a questionnaire or verbally asked.

  • How difficult was it to find the main menu?
  • What was the first impression when you opened the application?;
  • Did you feel in difficulty finding the new product list?
  • What seemed the easiest to you?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".
References

  • Handbook of Usability Testing by Jeffrey Rubin;
  • A Practical Guide to Usability Testing by Joseph S. Dumas and Janice C. Redish.
Other Notes
  • Users with less experience in using mobile apps have done the tasks in more time.
  • Users with more experience in using mobile apps were excited about the first interaction with the application and were curious to use it.

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • When 50% of users had a quick interaction without any difficulty with the mobile application
  • When 50% of users easily add desired products to shopping cart.
  • When 50% of users entered in less than 50 seconds to see recommendations.
  • When 50% of users have accessed the social network to see what their friends have bought.
  • When 50% of users have easily used QR code to scan products.
  • When 50% of users have chosen their own preferences or allergies.

Summary

  • Finding 1: User 3 finds it difficult to use the app as the font cannot increase.
  • Finding 2: All 4 users successfully completed 3 tasks.
  • Finding 3: A no of 2 users feel the app is not fully modern.
  • Finding 4: A no of 2 users feel the coloring schema can be more lively.

Post-test Questions Answers
  • From a scale of 1-10 how likely are you to use the application?
  • Did you find the app unnecessarily complex?
  • Would you need support from a more technically advanced person prior to using the application?
  • Do you feel the intro tutorial teaches you enough?
Test Observations
  • User 3 had difficulties in seeing the font as she wore grasses, a mechanism of increasing the font size would suffice.
  • User 1 and 4 believed the design can be improved with more recent android material design components.
  • User 1 and 4 believed that the app needs to have a more colorful palette.
  • Users need some time to accommodate with the concept and app but they understand what the purpose is and how to use it.
Potential Design Improvements
  • Improve design color schema
  • Make the app more up to material design
  • Add mechanism for increasing font size
